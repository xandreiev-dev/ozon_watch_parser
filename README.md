# Ozon Watch Parser

Парсер умных часов с Ozon для подготовки XLSX-выгрузки под дальнейшую обработку в watch matcher и запись в БД. Проект работает через реальный Chrome в режиме CDP, выставляет регион Москва, обходит брендовые листинги и собирает карточки прямо из открытой страницы Ozon.

## Главное

- CDP включен по умолчанию: `http://localhost:9222`.
- Если Chrome с CDP уже открыт, парсер подключается к нему и не закрывает его после работы.
- Если CDP недоступен, парсер может сам открыть Chrome с `--remote-debugging-port=9222`.
- Данные пишутся потоково в брендовые XLSX-файлы.
- За один общий запуск парсер обходит все заданные брендовые ссылки.
- После обхода формируется итоговый файл `Ozon_watch_ru_YYYYMMDD.xlsx`.
- Формат включает базовые колонки Avito-парсера и дополнительные поля Ozon из листинга.
- По умолчанию используется два прохода на бренд: исходная ссылка и `sorting=price`.

## Установка

```bash
pip install -r requirements.txt
playwright install chromium
```

## Ссылки по брендам

Ссылки задаются в [config.toml](C:/work/ozon_watch_parser/config.toml).

В блок `[urls]` добавляется одна ссылка на листинг каждого бренда. Пустые бренды пропускаются.

```toml
[urls]
apple = "https://www.ozon.ru/..."
samsung = "https://www.ozon.ru/..."
vivo = "https://www.ozon.ru/..."
```

Ссылки должны быть на `www.ozon.ru`. Если случайно вставить `ozon.kz`, загрузчик конфига приведёт домен к `www.ozon.ru`, но лучше сразу копировать ссылки после выставления региона Москва.

Основные настройки парсинга:

```toml
[parser]
pages = 20
min_cards = 200
min_price = 1000
max_price = 300000
include_rating_sort = false
```

Параметры `min_price` и `max_price` отсекают только товары вне диапазона цены. Словесная фильтрация по названиям вроде ремешков/браслетов не применяется, чтобы не потерять спорные позиции.

По умолчанию парсер делает два обхода на бренд: исходная ссылка и вариант с `sorting=price`. Третий обход с `sorting=rating` можно включить через `include_rating_sort = true` или CLI-флаг `--include-rating-sort`, но он заметно увеличивает время полного прогона.

## Регион Москва

После первого открытия Ozon парсер выставляет Москву через интерфейс сайта: задаёт geolocation Москвы, открывает виджет адреса, вводит `141-99-90` и подтверждает адрес доставки. Это нужно, чтобы цены, доставка и количество товаров собирались для московского региона, а не для случайной геолокации текущей сессии.

## Пошлина

В итоговый файл добавляется поле `tax_price`. Расчёт используется только для товаров, которые похожи на зарубежную поставку Ozon Global: учитываются признаки в названии, срок доставки и разница между `Price` и `Discount Price`. Локальные признаки вроде `Ростест`, `EAC`, `РСТ` и `Российская версия` отключают расчёт.

Формула: `15%` от суммы выше порога `200 EUR` + таможенный сбор Ozon `689 руб`. Курс EUR берётся из CBR JSON API, при недоступности API используется fallback `100 руб`.

## Формат XLSX

Итоговый файл формируется один: `Ozon_watch_ru_YYYYMMDD.xlsx`.

Первые колонки совпадают с форматом Avito-парсера:

```text
Название, Цена, URL, Описание, Дата публикации, Продавец, Адрес,
Адрес пользователя, Координаты, Изображения, Поднято, Звезды,
Отзывы, Доставка
```

Дальше идут дополнительные поля Ozon/watch: `brand`, `model`, `condition`, `size`, `color`, `warranty`, `Article`, `is_global`, `tax_price`, `Price`, `Discount Price`, `source_brand` и другие технические поля.

Поле `Описание` при листинговом сборе обычно пустое: Ozon не отдаёт описание в карточке листинга. Для заполнения описаний нужен отдельный медленный режим с заходом в каждую карточку товара.

## Рекомендуемый запуск через Chrome CDP

```bash
chrome.exe --remote-debugging-port=9222
python -m ozon_watch_parser.cli --once
```

В рабочей папке проекта можно запускать через обёртку:

```bash
python run_parser.py --once
```

Парсинг одного бренда:

```bash
python run_parser.py --once --brand apple
```

Контрольный полный прогон с дополнительным обходом `sorting=rating`:

```bash
python run_parser.py --once --include-rating-sort
```

Запуск без CDP:

```bash
python run_parser.py --once --no-cdp
```

## Структура

```text
src/ozon_watch_parser/
  browser/      CDP-first подключение к Chrome
  config/       бренды, TOML-конфиг, URL листингов, колонки
  domain/       нормализация часов
  export/       потоковый XLSX и агрегация
  ozon/         JS extractor и прокрутка листинга
  services/     сценарий парсинга брендов
  utils/        даты, URL, пошлина
```
